## 项目概述

**车牌号**: 粤BCW7826  
**分析日期**: 2025-09-25, 2025-09-26  
**项目目标**: 基于新能源出租车运营数据，完成车辆4种核心运营状态的全时序识别与完整描述

## 四种运营状态定义

| 状态名称 | 核心业务含义 |
|---------|-------------|
| occupied（载客） | 有乘客的运营状态，从乘客上车到下车的订单全时段 |
| heading（前往充电站） | 载客订单结束后，到充电开始前，车辆从订单终点向充电站定向行驶的时段 |
| recharging（充电） | 车辆静止在充电站、完成电量补给的时段 |
| cruising（空驶巡游） | 非载客、非前往充电、非充电的时段，司机空驶寻找乘客的运营时段 |

## 数据预处理流程

### 1. 字段解析与结构化转换
- 解析原始txt文件的17个字段
- 将毫秒级Unix时间戳转换为北京时间
- 统一时间格式和数据类型

### 2. 数据清洗流程
1. **过滤未完成订单**: 只保留order_status=0的有效订单
2. **剔除缺失值**: 核心字段（时间、经纬度）缺失的记录直接剔除
3. **校验时间逻辑**: 确保下车时间 > 上车时间
4. **校验地理位置**: 确保经纬度在深圳运营范围内（经度113.7-114.5，纬度22.4-22.8）
5. **校验订单里程**: 剔除0.5-100公里范围外的异常订单
6. **校验订单时长**: 剔除1分钟-2小时范围外的异常订单
7. **去重处理**: 基于时间戳和经纬度去重

### 3. 清洗结果统计

## 状态识别算法

### 1. 载客状态 (occupied)
- **判定规则**: 处于有效订单的「上车时间-下车时间」区间内
- **数据来源**: 清洗后的有效订单数据

### 2. 充电状态 (recharging)
- **判定规则**: 
  - GPS经纬度连续≥30分钟浮动≤0.0005（约50米内）
  - 静止时段内无任何有效订单
  - 静止时长≥30分钟
- **数据来源**: GPS时序数据

### 3. 前往充电状态 (heading)
- **判定规则**:
  - 处于「充电前最后一笔订单下车时间」至「充电开始时间」之间
  - 时段内无有效载客订单
  - 行驶时长≤60分钟
- **数据来源**: 订单数据、充电时间边界

### 4. 空驶巡游状态 (cruising)
- **判定规则**:
  - 处于「上一笔订单下车时间」至「下一笔订单上车时间/heading开始时间」之间
  - 时段内无载客、无充电、无前往充电站的定向行驶
- **数据来源**: 订单时序数据、状态时间边界

## 司机行为模式总结

### 1. 运营时间规律
- 司机主要在夜间和凌晨时段运营
- 存在明显的充电休息时段
- 运营时间跨度大，覆盖全天24小时

### 2. 充电行为习惯
- 倾向于在订单完成后前往充电站
- 单次充电时长通常在30-60分钟
- 9月26日充电频率更高（4次 vs 2次）

### 3. 运营效率特征
- 日均订单量约36笔
- 平均订单里程4-5公里
- 空驶巡游时间占比较高（60-70%），存在优化空间
- 载客率约24-26%
